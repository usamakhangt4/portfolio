## Problem

Issue trackers are common learning projects, but they can still show useful backend-connected UI patterns when the code includes validation, persistence, and route structure.

## Approach

This project uses Next.js App Router, API routes, Prisma schema/migrations, and Zod validation. The interface uses Radix Themes components and a simple issue workflow.

## Outcome

It is best kept as supporting work. It proves some useful full-stack patterns, but it should not be positioned ahead of the stronger libraries, developer tool, medicine search UI, or computer vision studio.
